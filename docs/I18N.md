# Internationalization (I18n)

## Overview
The GameStore Frontend supports English (`en`) and Bahasa Indonesia (`id`) using a **hybrid I18n architecture**. Translation strings are resolved on the server for speed and SEO, while a client-side context ensures interactive components (like the Navbar and Cart) stay in sync.

## Implementation Details

### 1. Dictionaries (`lib/i18n/translations.ts`)
A centralized object containing all localized strings. All keys are mirrored across both supported languages.

### 2. Server-Side Utility (`lib/i18n/get-translations.ts`)
A pure function for **Server Components**. It retrieves the dictionary synchronously based on the `lang` parameter.

```tsx
// Server Component Usage
import { getTranslations } from '@/lib/i18n/get-translations';

export default async function Page({ params }) {
  const { lang } = await params;
  const { t } = getTranslations(lang);
  return <h1>{t('common.welcome')}</h1>;
}
```

### 3. Client-Side Context (`lib/i18n/i18n-context.tsx`)
A `Provider` that wraps the application layout. It is initialized from the server-fetched `lang` and provides the `t()` function to **Client Components**.

```tsx
// Client Component Usage
'use client';
import { useI18n } from '@/lib/i18n/i18n-context';

export function SearchButton() {
  const { t } = useI18n();
  return <button>{t('common.search')}</button>;
}
```

### 4. Routing & Persistence
- **Language Prefixes:** All routes are nested under `/[lang]/`.
- **Preference Persistence:** When a user switches language via the Navbar, the `_preferred-lang` cookie is updated, and `router.push` is used to navigate to the new path.
- **Server Proxy:** A `proxy.ts` layer handles redirects for root requests (`/`) to the appropriate localized path based on the preference cookie or browser headers.

## Adding New Translations
1. Add the new key and its English value to `translations.en` in `lib/i18n/translations.ts`.
2. Add the corresponding Indonesian value to `translations.id`.
3. The type system (TypeScript) ensures that keys are consistent across both objects.
4. Run `bun run test` to verify key parity across dictionaries.
